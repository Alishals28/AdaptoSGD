**Department of Computing**
**Course: Parallel and Distributed Computing**
**Semester Project Assignment Brief**
**Group Size: 3 Students**

**1. Purpose**

The core KPI of this project is the quality of distributed systems design and analysis. Application logic alone is insufficient.

**2. Learning Outcomes**

By completing this project, students should be able to:

- Critically evaluate distributed system architectures.

- Analyze scalability, performance, and fault tolerance trade-offs.

- Design a distributed solution grounded in research literature.

- Implement and experimentally validate a system-level improvement.

- Justify novelty and system contribution at C-6 level.

**3. Project Themes (Select One)**

Each group must choose one of the following themes:

- Custom Consensus Protocol under Network Partitions

- Adaptive Distributed Load Balancer

- GPU vs CPU Parallel Numerical Solver (Performance Modeling Focus)

- Mini Pregel: Parallel Graph Processing Engine

- Distributed SGD: Parameter Server vs Ring AllReduce

- Gossip vs Structured Overlay Aggregation

- Byzantine-Resilient Distributed Averaging

- Distributed Checkpointing Strategy Optimization

- Distributed Cache Coherence Simulator

- Serverless Execution Simulator with Adaptive Resource Policy

Any variation must receive prior approval.

**4. Mandatory Academic Requirements**

**4.1 Literature Review**

Minimum 10 peer-reviewed research papers (conference/journal).
The review must include:

- Comparative summary table

- Identified limitations

- Explicit gap analysis

**4.2 Gap Analysis**

Students must clearly define:

- What problem remains unresolved?

- What tradeoff is insufficiently addressed?

- What failure or scalability bottleneck exists?

- What improvement is proposed?

Projects without a defensible gap will not proceed.

**5. Deliverables**

***Deliverable 1: ******Assignment 1***** ****Proposal Defense (****Week starting form 02 March****)**

Presentation and project proposal (10–12 minutes) including:

- Problem statement

- Distributed systems challenge

- Literature review summary

- Gap analysis

- Proposed hypothesis

- Performance metrics

- Experimental plan

Evaluation focus: clarity of distributed core and research grounding.

**Deliverable 2: **** *****Assignment ******2 *****Full System Design Document **

The design report must include:

- System architecture diagram

- Communication model

- Consistency model (if applicable)

- Failure model

- Performance modeling (Amdahl, Gustafson, or message complexity)

- Scalability assumptions

- Implementation plan

- Risk analysis

This stage assesses analytical depth (C-5 level).

**Deliverable 3:**** *****Project:***** Final Presentation ****&**** Demonstration **

Must include:

- Experimental setup description

- Workload modeling

- Scalability results

- Failure scenario evaluation

- Comparative baseline analysis

- Sensitivity analysis

- Measurable improvement

- C-6 novelty justification

Working software without rigorous evaluation will not receive high grades.

**6. C-6 Requirement (Create Level)**

To qualify for highest performance band, students must:

- Propose at least one novel system-level improvement

- Implement and test it

- Compare against baseline

- Demonstrate measurable improvement

- Analyze trade-offs

**7. Evaluation Scheme**

| **Component** | **Weight** |
| --- | --- |
| Literature Review & Gap Analysis | 10% |
| Proposal Defense | 10% |
| System Design Quality | 20% |
| Experimental Methodology | 20% |
| Results & Analysis | 20% |
| Novelty & C-6 Justification | 20% |

**8. Constraints**

- Projects must demonstrate distributed or parallel behavior across multiple processes, threads, or nodes.

- Pure theoretical or purely machine learning projects without distributed evaluation are not permitted.

- All experimental results must be reproducible.

- Code and report submission is mandatory.

**9. Academic Integrity**

All sources must be cited properly.
Plagiarism or uncredited reuse of open-source code without modification will result in zero marks.
